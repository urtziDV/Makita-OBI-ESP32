# User Manual - System Operation

Welcome to the **Makita OBI ESP32** diagnostic tool. This manual explains how to get the most out of the interface.

## 1. Initial Connection

1. Connect the device to the battery or power supply.
2. From your smartphone or PC, connect to the WiFi network: **`Makita_OBI_ESP32`**.
3. Open your browser and go to: **`http://makita.local`** or **`http://192.168.4.1`**.

## 2. Reading Data

- **"Read Info" Button**: Performs a full read of the battery (Model, Cycles, Lock Status, Initial Voltages).
- **"Read Voltages" Button**: Updates only the cell voltages (Quick use).
- **"Reset Errors" Button**: Attempts to clear persistent errors and unlock the controller.
- **Auto-read**: If you activate the "Real-time Updates" switch, the system will automatically refresh the data every 3 seconds.
- **Imbalance HUD**: In the header, you will see a badge indicating if the pack is **Balanced**, has **Imbalance**, or is in a **Critical** state.

## 3. History Charts

- Below the cell status, you will see a real-time chart.
- Each line represents a cell (1 to 5). On BL14 batteries, only 4 lines will automatically be shown.
- This chart is useful to see if a cell's voltage drops faster than the others when the battery is under load.

## 4. WiFi Configuration (Workshop Mode)

To connect the device to your regular WiFi network:

1. Click the gear icon (**⚙️**).
2. In the "WiFi Configuration" section, enter the **SSID** (Network Name) and **Password**.
3. Click "Connect to WiFi".
4. The ESP32 will restart. You can now access it from any device on your local network using **`http://makita.local`**.

## 5. Health Interpretation (SOH)

- **Excellent**: New battery or with very little use.
- **Fair**: Normal use, slightly reduced capacity.
- **Degraded**: A balancing cycle is recommended, or check individual cells.
- **Locked**: The BMS has detected a critical failure and locked the battery.

## 6. Troubleshooting

| Problem | Probable Cause | Solution |
| :--- | :--- | :--- |
| WiFi does not appear | Pin shorted or Serial conflict. | Make sure you moved ENABLE to Pin 5. |
| "(Empty Bus)" appears | Bad physical connection. | Check that the data wire makes good contact with the center pin. |
| Web page doesn't load | Filesystem missing. | Run "Upload Filesystem Image" from PlatformIO. |
| Voltages at 0.00V | Battery asleep or non-existent cell. | Click "Read Info". In BL14 the 5th cell is always empty. |
